# 习惯打卡应用 — 功能增强 PRD

## 1. 项目信息

| 字段 | 内容 |
|------|------|
| **项目名称** | habit_tracker_enhancement |
| **基础栈** | Next.js 16 + Prisma (SQLite) + NextAuth + Tailwind CSS |
| **当前功能** | 用户注册/登录、添加/修改/删除习惯、每日打卡/取消打卡、进度条、连续天数统计、月度柱状图、最近 7 天可视化 |
| **原始需求** | 基于竞品调研（10+ 主流习惯打卡应用），补齐 5 个缺失的关键功能：丰富统计、习惯分组、暗色模式、数据导出、打卡备注 |

## 2. 产品定义

### 2.1 产品目标

1. **提升数据洞察力** — 为用户提供多维度的统计数据（完成率、最佳连续天数、日历热力图），让进步可视化，增强坚持动力
2. **优化组织管理能力** — 支持习惯按标签/类别分组，帮助用户结构化地管理大量习惯
3. **提升用户体验舒适度** — 增加暗色模式、打卡备注功能，降低长时间使用的视觉疲劳，增加打卡意图的记录能力

### 2.2 用户故事

| # | 用户故事 |
|---|---------|
| US1 | As a 活跃用户, I want **查看每个习惯的完成率和最佳连续天数** so that **我能更清楚地了解自己的长期进步趋势** |
| US2 | As a 打卡种类多的用户, I want **按标签/分组管理我的习惯** so that **我能快速筛选和查看特定类别的习惯（如"健康"、"学习"、"工作"）** |
| US3 | As a 夜间使用者, I want **一键切换暗色模式** so that **在低光环境下使用时眼睛更舒适** |
| US4 | As a 数据爱好者, I want **将我的打卡记录导出为 CSV 文件** so that **我可以在其他工具中做深入分析或备份数据** |
| US5 | As a 注重细节的用户, I want **每次打卡时写下简短备注** so that **我记录当天的具体感受或特殊情况** |

## 3. 技术规范

### 3.1 需求池

#### P0 — 必须实现（核心体验缺陷）

| ID | 需求 | 描述 |
|----|------|------|
| P0-1 | **日历热力图** | 在统计页面新增日历热力图组件，用颜色深浅直观展示一年中每天的打卡状态（类似 GitHub 贡献图），必须支持按习惯切换查看 |
| P0-2 | **完成率统计** | 在统计页面展示每个习惯的整体完成率（已打卡天数 ÷ 总天数 × 100%），以及指定时间范围（周/月）的完成率 |
| P0-3 | **最佳连续天数** | 在统计页面展示每个习惯的历史最佳连续打卡天数（best streak），与当前连续天数并列显示 |
| P0-4 | **习惯分组（标签）** | 用户可为习惯设置标签/分组名称（如"健康"、"学习"、"工作"），在仪表盘支持按分组筛选和分组折叠展示，创建/编辑习惯时可选或新建分组 |

#### P1 — 应该实现（重要但不阻塞）

| ID | 需求 | 描述 |
|----|------|------|
| P1-1 | **暗色模式** | 使用 CSS 变量实现完整的暗色主题，所有页面（仪表盘、统计页、登录/注册）均适配；在 Header 增加切换开关，偏好持久化到 localStorage |
| P1-2 | **数据导出（CSV）** | 在统计页面增加"导出数据"按钮，按习惯导出打卡记录为 CSV 文件（包含日期、打卡状态、备注等字段） |
| P1-3 | **打卡备注** | 打卡时为每个 checkin 关联一个可选的短文本备注字段（最多 100 字），打卡后可在习惯详情中查看历史备注；数据库 Checkin 模型增加 note 字段 |

#### P2 — 可以延后（锦上添花）

| ID | 需求 | 描述 |
|----|------|------|
| P2-1 | **月度完成率** | 在统计页的月度柱状图下方或 Tooltip 中增加月度完成率（当月打卡天数 ÷ 当月天数）的显示 |
| P2-2 | **分组管理页面** | 独立的分组管理页面/弹窗，支持 CRUD 分组及其颜色标签 |
| P2-3 | **导出全部数据** | 支持一键导出所有习惯的完整打卡记录为一个 CSV 文件 |

### 3.2 UI 设计稿草案

#### 全局
- **布局不变**：保持当前 `max-w-2xl` 居中布局 + 顶部 sticky Header 结构
- **暗色模式**：通过 `class` 策略（Tailwind `dark:` 前缀），CSS 变量定义色板，localStorage 存储偏好

#### 仪表盘（Dashboard）
- 习惯列表展示保持不变，新增 **分组筛选器**（横向滚动标签栏，类似 iOS 分类选择器）
- 选择分组后，习惯列表只展示该分组下的习惯
- 未分组的习惯显示为"未分类"

#### 统计页（Stats）
- 现有组件保留（当前连续天数、累计打卡次数、最近 7 天、月度柱状图）
- **新增区块**：
  - **最佳连续天数卡片** — 与"当前连续天数"并列展示
  - **整体完成率** — 百分比进度环或进度条
  - **日历热力图** — 采用类似 GitHub 贡献图布局（横轴月份，纵轴星期），色阶表示当天打卡状态

#### 打卡弹窗/行内
- 点击"打卡"按钮后，弹出简短的备注输入框（可选），输入后确认打卡；或直接在习惯卡片上展开备注输入区域
- 打卡后，当天已打卡状态下显示备注内容预览

#### 新建/编辑习惯弹窗
- 在现有表单（名称、描述、颜色）基础上，增加 **分组选择器**：
  - 下拉选择已有分组，或输入新分组名称自动创建
  - 分组可附带颜色标记（可选）

### 3.3 待确认问题

1. **日历热力图的展示范围**：展示最近 365 天还是按自然年展示？建议按滚动 365 天以保持始终有数据。
2. **分组归属关系**：一个习惯是否可属于多个分组（多对多）？建议先实现一个习惯只能归属一个分组（一对多），设计更简单。
3. **备注保留期限**：备注是否需要永久保留，还是只保留最近 N 条？建议永久保留，数据量不大。
4. **CSV 导出格式**：导出的 CSV 是否需要包含所有习惯还是按习惯逐个导出？建议先实现按习惯逐个导出（P1），再考虑批量导出（P2）。
5. **暗色模式默认值**：首次访问的用户，暗色模式默认跟随系统（`prefers-color-scheme`）还是默认关闭？建议跟随系统偏好。
